# Data Model: Public Status Page and Network Probe Trends

## Public System Visibility

Represents the opt-in public exposure state for a Beszel system.

**Fields**

- `system_id`: relation to `systems`; unique; required.
- `public_enabled`: boolean; default `false`.
- `public_name`: optional text override for public display.
- `show_cpu`: boolean; default `true`.
- `show_memory`: boolean; default `true`.
- `show_disk`: boolean; default `true`.
- `created`: timestamp.
- `updated`: timestamp.

**Relationships**

- Belongs to one `System`.
- Read by public status composition only when `public_enabled = true`.

**Validation Rules**

- `system_id` must reference an existing system.
- Public name must be escaped in UI and cannot enable HTML/script execution.
- Existing systems are private until this record is enabled.

**State Transitions**

- `private` -> `public`: admin enables `public_enabled`.
- `public` -> `private`: admin disables `public_enabled`; anonymous views stop showing the system after refresh.

## Public Status Summary

Sanitized read model returned to anonymous visitors.

**Fields**

- `id`: public-safe system identifier.
- `name`: public display name.
- `status`: `up`, `down`, `paused`, or `stale`.
- `freshness`: timestamp or relative freshness marker.
- `cpu_percent`: optional number.
- `memory_percent`: optional number.
- `disk_percent`: optional number.
- `network_probes`: public-safe probe summaries and chart series when visible.

**Relationships**

- Composed from `systems`, latest system info, public visibility settings, and visible probe results.

**Validation Rules**

- Must not include host, port, internal address, users, tokens, owner-only IDs, SSH details, or admin actions.
- Must include only systems with public visibility enabled.

## Network Probe

Administrator-defined probe target executed by selected nodes.

**Fields**

- `id`: unique identifier.
- `name`: required display label.
- `type`: enum `tcping`, `icmp_ping`, `http_get`.
- `target`: required target string.
- `interval_seconds`: integer; required; minimum enforced during planning/implementation.
- `timeout_seconds`: integer; required; less than interval.
- `public_visible`: boolean; default `true`.
- `enabled`: boolean; default `true`.
- `execution_mode`: enum `automatic`, `advanced`; default `automatic`.
- `created`: timestamp.
- `updated`: timestamp.

**Relationships**

- Has many `Network Probe Results`.

**Validation Rules**

- No built-in targets are created by default.
- Probe count is not limited to three.
- `target` format depends on `type`:
  - `tcping`: host/IP plus port.
  - `icmp_ping`: host/IP.
  - `http_get`: HTTP or HTTPS URL.
- `public_visible = false` hides this probe from anonymous public views even when assigned systems are public.
- `execution_mode = automatic` means the UI does not require explicit agent binding in the primary flow.

**State Transitions**

- `enabled` -> `disabled`: stop scheduling/executing new results; historical results remain readable to authorized users.
- `public_visible` -> `hidden`: anonymous views stop showing the probe after refresh.

## Network Probe Assignment

Maps a probe to nodes that execute it.

**Fields**

- `probe_id`: relation to `Network Probe`; required.
- `system_id`: relation to `systems`; required.
- `enabled`: boolean; default `true`.
- `created`: timestamp.
- `updated`: timestamp.

**Relationships**

- Belongs to one `Network Probe`.
- Belongs to one executing system.
- Has many `Network Probe Results` through `probe_id` + `system_id`.

**Validation Rules**

- `(probe_id, system_id)` must be unique.
- Assigned system must represent an agent-capable system.
- Disabled assignment stops new checks for that system/probe pair.

## Network Probe Result

Timestamped measurement generated by an executing agent node.

**Fields**

- `probe_id`: relation to `Network Probe`; required.
- `system_id`: relation to executing system; required.
- `type`: enum copied from probe at execution time.
- `target`: target copied from probe at execution time.
- `success`: boolean.
- `latency_ms`: optional number; present for successful latency-capable checks.
- `packet_loss_percent`: optional number; primarily for ICMP.
- `http_status`: optional integer; for HTTP GET.
- `error`: optional short error category/message safe for authenticated users.
- `created`: timestamp.
- `bucket`: optional aggregation bucket such as `1m`, `10m`, or `120m` if retention/aggregation mirrors system stats.

**Relationships**

- Belongs to one `Network Probe`.
- Belongs to one executing `System`.

**Validation Rules**

- Result must identify the executing system.
- Failed checks still create a result with `success = false`.
- One failed probe result must not block other probes or agents.

## Latency Trend Window

Requested chart time window for displaying probe history.

**Fields**

- `range`: enum aligned with supported chart windows such as recent, daily, and longer-range views.
- `step`: derived display interval.
- `probe_ids`: selected probes.
- `system_ids`: selected executing systems.

**Validation Rules**

- Anonymous windows only include public systems and probes where public display is allowed.
- Authenticated windows follow existing user/system authorization rules.

## Public Page

Frontend route that renders public status summaries.

**Fields**

- `systems`: list of `Public Status Summary`.
- `generated_at`: timestamp.
- `empty_state`: boolean derived from `systems.length === 0`.

**Validation Rules**

- Must be accessible without login and serve as the home route at `/`.
- Must not provide admin actions.
- Must not reveal private systems through errors, empty states, or timing-specific messages.
